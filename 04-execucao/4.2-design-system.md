## Instruções para o Claude Design

```
Crie um design system chamado "VeredaTech" para um produto B2G de gestão de
indicações parlamentares (câmaras municipais). Direção: clean, leve, moderno.
Base neutra quente, cartões brancos flutuando com sombra suave, TODOS os
elementos de interface (inputs, botões, chips, tags) em formato de pílula
(border-radius total). Cartões com raio 20px. CTA primário sempre em preto
(quase-preto). Cor de acento (verde-floresta escuro) reservada para estado
"positivo/respondido" e ações de marca — não para decoração geral.

TOKENS DE COR (light):
- bg (fundo de tela): #EFEEEA
- surface (cartões, inputs, painéis): #FFFFFF
- ink (texto principal, CTA primário): #1A1B1E
- muted (texto secundário, rótulos): #6D7078
- line (bordas sutis, divisores): #E3E1DB
- accent / accent-emphasis (marca, estado "respondida"): #1E5F4E
- accent-soft (fundo de chips/realces do acento, anel de foco): #E4EFEA
- warning (estado "em silêncio"): #B45309 · warning-soft: #FCEFDD
- danger (estado "prazo regimental vencido"): #C22F2F · danger-soft: #FBE7E5

TOKENS DE COR (dark) — mesma estrutura, invertendo luminância sem inverter
literalmente as cores (accent, warning, danger mantêm a mesma identidade,
ajustando saturação/luminosidade para contraste em fundo escuro):
- bg: #17181B · surface: #202226 · ink: #ECEBE7 · muted: #9A9CA3 · line: #33353B
- accent permanece #1E5F4E (ou versão levemente clareada se o contraste sobre
  #202226 exigir) · accent-soft escurecido para #1B2A24

TIPOGRAFIA:
- Display/títulos: Schibsted Grotesk, pesos 500/600/700, letter-spacing
  levemente negativo em títulos grandes (-0.02em)
- Corpo e UI: Inter, pesos 400/500/600
- Dados/números tabulares: Inter com font-variant-numeric: tabular-nums
- Escala: 12 / 13 / 14 / 15 / 16 / 20 / 26 / 32 / 40px — usar só esses degraus

ESPAÇAMENTO: escala em base 4px — 4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64px.

RAIO: pílula (999px) para todo controle interativo (input, botão, chip,
select, tag, avatar). 20px para cartões. 28px para painéis/containers
grandes. 12-16px para blocos menores dentro de um cartão (ex.: item de lista).

SOMBRA: cartões usam sombra dupla suave — 0 1px 2px rgba(26,27,30,.05) + 0
8px 24px rgba(26,27,30,.06). Nada mais pesado que isso; sem sombras duras.

COMPONENTES BASE (com todos os estados: default, hover, focus, disabled,
e onde fizer sentido: loading, error):
1. Botão primário (preto, texto branco, pílula)
2. Botão secundário (fundo surface, borda line, texto ink, pílula)
3. Botão de marca/acento (fundo accent, texto branco, pílula) — só para
   ações que "fecham o ciclo" ou geram artefato de marca (ex.: gerar balanço)
4. Input de texto (pílula, borda line, foco = borda accent + anel
   accent-soft de 3px)
5. Select (mesma base do input, indicador de seta discreto)
6. Chip de estado — 3 variantes: ok (accent/accent-soft), warning
   (warning/warning-soft), danger (danger/danger-soft) — bolinha + rótulo
7. Card padrão (surface, raio 20px, sombra suave, padding 20-24px)
8. Item de lista/timeline (borda line, raio 12-16px, sem sombra — mais leve
   que o card)
9. Navegação superior (cabeçalho fixo: símbolo + wordmark à esquerda,
   ações à direita)
10. Barra de progresso/indicador percentual (trilha em line, preenchimento
    em accent, pílula)

REGRAS DE USO:
- Cor semântica (warning/danger) é reservada a ESTADO do ciclo da indicação,
  nunca usada como cor decorativa de marca.
- Foco sempre visível (WCAG): outline ou anel de 2-3px, nunca remover
  outline sem substituto.
- Um único elemento de destaque de acento por tela — evitar poluir com
  verde em todo canto.
- Ícones: traço único, terminais arredondados, mesma linguagem do símbolo
  da marca (4.1) — não misturar com bibliotecas de ícone de cantos vivos.
```

## Referência rápida (para uso direto no protótipo HTML da 4.4)

Estes são os mesmos tokens em CSS custom properties, prontos para o `global.css` do protótipo:

```css
:root{
  --bg:#EFEEEA; --surface:#FFFFFF; --ink:#1A1B1E; --muted:#6D7078; --line:#E3E1DB;
  --accent:#1E5F4E; --accent-soft:#E4EFEA;
  --warning:#B45309; --warning-soft:#FCEFDD;
  --danger:#C22F2F; --danger-soft:#FBE7E5;
  --radius-pill:999px; --radius-card:20px; --radius-panel:28px; --radius-sm:14px;
  --shadow-card: 0 1px 2px rgba(26,27,30,.05), 0 8px 24px rgba(26,27,30,.06);
  --space-1:4px; --space-2:8px; --space-3:12px; --space-4:16px; --space-5:20px;
  --space-6:24px; --space-7:32px; --space-8:48px; --space-9:64px;
  --font-display:"Schibsted Grotesk", sans-serif;
  --font-body:"Inter", sans-serif;
}
```

## Pendência
Nenhuma — tokens e componentes prontos para uso direto na 4.3/4.4. Fontes reais (Google Fonts) a baixar/linkar como `@font-face` local no protótipo, respeitando a regra de "só front-end, sem dependência externa em runtime" quando aplicável.
